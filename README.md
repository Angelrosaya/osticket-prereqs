<h1>osTicket - Prerequisites and Installation</h1>
This tutorial outlines the prerequisites and installation of the open-source help desk ticketing system osTicket.<br />


<h2>Environments and Technologies Used</h2>

- Microsoft Azure (Virtual Machines/Compute)
- Remote Desktop
- Internet Information Services (IIS)

<h2>Operating Systems Used </h2>

- Windows 10 Enterprise Gen 2</b> (22H2)

<h2>List of Prerequisites</h2>

- Setup a Virtual Machine in Azure
- Install the osTicket requirements
- Install osTicket itself
- Do the after-installation configuration of osTicket
- Explore osTicket as a Help Desk Professional (Create, interact, close tickets)

<p>

  - First you want to go to the Microsft azure page and create a free trial that enables you to use 200 free credits that will get you through these labs.
- Go to virtual machines and create a new vm.
- Create a new resource group by clicking Create new in blue lettering and name it "osTicket" or whatever you want to call it, for our virtual machine name make it "osticket-vm" to keep it simple but you can choose what you want, choose a region; for me I chose "East US 2", choose your image; for this lab we will be using Windows 10 enterprise Gen2 (22H2), select your size anything with 4CPUs will do just fine.
- Create a Username and Password, for this we will use "labuser" for our username and "osTicketPassword1" for our password.
- Make sure to click "I confirm I have an eligible Windows 10/11 license with multi-tenant hosting rights". If you don't at the end when it is time to click create you will get an error.
- Once done click review and create, wait for validation passed on the top left hand corner to pop up in green then click create and wait for the VM to deploy.
